# Cloud Technology: Modern Infrastructure

## 1. The Shift to "Cloud-Native"
Building applications specifically *for* the cloud rather than just moving old servers (Lift & Shift).
*   **Microservices:** Breaking large apps into small, independent pieces.
*   **Containers (Docker/Kubernetes):** Ensuring code runs the same way on any machine.

## 2. FinOps: Financial Operations
As cloud bills grow, companies are adopting **FinOps**—a cultural practice where engineers and finance teams collaborate to optimize cloud costs and maximize value.

## 3. Edge Computing & IoT
Processing data closer to where it's generated (e.g., in a factory, a self-driving car, or a hospital) rather than sending everything to a central data center.
*   **Latency:** Crucial for real-time applications where every millisecond counts.

## 4. Multi-Cloud & Hybrid Cloud
*   **Multi-Cloud:** Using multiple providers (AWS + Azure + Google) to avoid vendor lock-in and increase reliability.
*   **Hybrid Cloud:** Combining private servers (On-Premise) with public cloud for security and flexibility.

## 5. Sovereign Cloud
A growing movement, especially in the EU, for "Data Sovereignty"—ensuring that data is subject to the laws and control of the country where it is generated.
